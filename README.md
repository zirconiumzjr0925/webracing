# 霓虹弯道冲刺

一个使用 `HTML + CSS + 原生 JavaScript + Three.js` 实现的 3D 浏览器赛车游戏。

项目目标是用尽量轻量、直接可部署的方式，交付一个完整可玩的网页赛车原型，不依赖构建工具，不依赖复杂工程链路，适合直接作为静态站点部署。

## 项目特点

- 纯前端静态项目，直接由 `index.html`、`styles.css`、`game.js` 组成
- 基于 Three.js 的 3D 赛道、车辆、灯光与镜头系统
- 玩家可操控赛车，支持加速、刹车、转向、漂移、增压、复位
- 支持多名 AI 对手参与比赛，并按难度动态调整数量与强度
- 支持 3 圈竞速、顺序检查点计圈、实时排名、结算界面
- 支持多赛道、小地图、音效、道具系统、幽灵车系统、环境切换、本地成绩存档、调试面板
- 支持基于 Cloudflare Worker + Durable Objects 的在线排行榜与轻量联机房间
- 已适配 Cloudflare Pages 静态部署
- 已接入本地化低模环境资源与赛车模型，静态部署时不依赖外部模型热链

## 已实现功能

### 核心比赛

- 玩家赛车控制
- 3 圈完整比赛流程
- 闭环赛道与边界判定
- 顺序检查点计圈，防止抄近道偷圈
- 计时系统
- 实时排名系统
- 结算界面与成绩汇总

### 驾驶与物理

- 加速、减速、刹车、倒车
- 平滑转向
- 横向抓地与滑移
- 漂移机制
- 漂移蓄能与释放增压
- 出界自动复位
- 长时间逆行自动复位
- 玩家与 AI 车辆之间的碰撞反馈

### AI 对手

- AI 路径跟随
- 根据赛道弯曲程度自动调整速度
- 不同难度下动态增加 AI 数量与速度
- AI 卡住后的自动恢复
- AI 可拾取并自动使用道具

### 道具系统

- 赛道道具盒生成
- 玩家与 AI 可拾取道具
- 随机获得以下道具：
  - 加速道具：短时间额外提速
  - 锁定导弹：追击前车并造成减速
  - 地雷：放置在赛道上形成陷阱
  - 范围冲击波：减速附近对手
- HUD 显示当前持有道具

### 在线模式

- 开始界面支持 `单机 / 在线` 模式切换
- 在线模式可查看对应赛道与难度的在线排行榜
- 可创建房间、输入房间码加入房间、离开房间
- 房主可统一设置赛道、难度和环境，并开始比赛
- 远端玩家会作为半透明联机赛车同步到赛道和小地图中
- 完赛后会自动向 Cloudflare 在线排行榜提交成绩
- 在线服务默认地址：
  - `https://neon-bend-rush-service.zirconiumzjr.workers.dev/api`

### 幽灵车系统

- 记录玩家本局比赛轨迹
- 下一局生成半透明幽灵车
- 幽灵车不参与碰撞
- 用独立颜色区分正常赛车与幽灵车

### 环境系统

- 开始界面可选择环境：
  - 白天
  - 黄昏
  - 夜晚
- 使用光照、雾效、地面与赛道颜色变化实现不同氛围

### 环境美术资源

- 已本地化接入 `glTF / GLB` 低模环境模型
- 优先补强了树木、路锥、广告牌、看台、灯杆、护栏装饰、路边灌木、终点线横幅与旗帜
- 玩家车与 AI 车也已切换为本地化低模赛车模型视觉层
- 所有模型均使用相对路径加载，可直接随静态站点部署
- 资源许可与中文署名说明见 [CREDITS.md](./CREDITS.md)

### 成绩与存档

- 使用 `localStorage` 保存本地成绩
- 保存内容包括：
  - 最快单圈
  - 最佳总时间
  - 本地历史排行榜
- 结算界面显示历史最佳成绩
- 刷新纪录时显示“新纪录”提示

### 小地图与音效

- 左下角小地图显示当前赛道轮廓
- 小地图实时显示玩家、AI 与联机玩家位置
- 引擎声会随速度变化
- 漂移时有轮胎摩擦声
- 碰撞时有碰撞声

### 调试面板

- 内置开发调试面板
- 可实时调节：
  - `maxSpeed`
  - `acceleration`
  - `grip`
  - `driftFactor`
- 修改后立即作用于玩家车辆

## 操作方式

- `W` / `方向上`：加速
- `S` / `方向下`：刹车 / 倒车
- `A` / `方向左`：左转
- `D` / `方向右`：右转
- `Space`：漂移 / 手刹
- `F`：使用当前道具
- `C`：切换镜头
- `R`：手动复位

## 难度说明

开始界面可选择三档难度：

- `休闲`：AI 数量较少，整体节奏更平稳
- `进阶`：默认难度，竞争感更明显
- `极限`：AI 更多、更快，对路线和漂移要求更高

## 本地运行

这是一个纯静态项目，不需要安装依赖，也不需要构建。

由于项目使用了 ES Module，直接双击 `index.html` 在部分浏览器环境下可能会遇到模块加载限制。推荐通过任意静态服务器运行，或者直接部署到 Cloudflare Pages。

如果你使用的是支持静态预览的开发工具，也可以直接把仓库根目录作为站点根目录打开。

## Cloudflare Pages 部署

项目已经适配静态部署，可直接连接仓库进行发布。

推荐配置：

- Framework preset：`None`
- Build command：留空
- Build output directory：仓库根目录

部署后默认入口文件为 `index.html`。

## Cloudflare 在线服务部署

联机房间和在线排行榜由独立的 Cloudflare Worker 提供，代码位于 `cloudflare/worker.js`。

核心能力：

- 在线排行榜：按赛道与难度分组保存成绩
- 房间系统：创建、加入、离开房间
- 实时同步：通过 Durable Objects + WebSocket 广播玩家状态
- 存储：使用 SQLite Durable Objects 保存房间与排行榜状态

部署相关文件：

- `cloudflare/worker.js`
- `cloudflare/wrangler.toml`

如果前端和在线服务不在同一个域名下，开始界面的“在线服务地址”输入框可以改成你自己的 Cloudflare Worker 地址。

## 设计要点

### 为什么使用纯静态结构

- 便于快速预览和分享
- 无需 Node 构建流程
- 非常适合部署到静态托管平台

### 为什么将 Three.js 本地化

- 避免外部 CDN 不稳定导致加载失败
- 确保资源路径完全可控
- 便于 Cloudflare Pages 等平台直接部署

### 为什么将模型资源本地化

- 避免运行时依赖外部模型热链
- 保证 Cloudflare Pages 等静态托管场景下的稳定访问
- 便于统一管理许可、署名和资源轻量化替换

### 为什么采用圆角矩形赛道

- 更稳定，便于 AI 路径跟随
- 容易生成闭环中心线
- 更适合做进度投影、检查点和出界判定
