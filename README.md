# 霓虹弯道冲刺

一个使用 `HTML + CSS + 原生 JavaScript + Three.js` 实现的 3D 浏览器赛车游戏。

项目目标是用尽量轻量、直接可部署的方式，交付一个完整可玩的网页赛车原型，不依赖构建工具，不依赖复杂工程链路，适合直接作为静态站点部署。

## 项目特点

- 纯前端静态项目，直接由 `index.html`、`styles.css`、`game.js` 组成
- 基于 Three.js 的 3D 赛道、车辆、灯光与镜头系统
- 玩家可操控赛车，支持加速、刹车、转向、漂移、增压、复位
- 支持多名 AI 对手参与比赛，并按难度动态调整数量与强度
- 支持 3 圈竞速、顺序检查点计圈、实时排名、结算界面
- 支持道具系统、幽灵车系统、环境切换、本地成绩存档、调试面板
- 已适配 Cloudflare Pages 静态部署

## 已实现功能

### 核心比赛

- 玩家赛车控制
- 3 圈完整比赛流程
- 闭环赛道与边界判定
- 顺序检查点计圈，防止抄近道偷圈
- 计时系统
- 实时排名系统
- 结算界面与成绩汇总

### 驾驶与物理

- 加速、减速、刹车、倒车
- 平滑转向
- 横向抓地与滑移
- 漂移机制
- 漂移蓄能与释放增压
- 出界自动复位
- 长时间逆行自动复位
- 玩家与 AI 车辆之间的碰撞反馈

### AI 对手

- AI 路径跟随
- 根据赛道弯曲程度自动调整速度
- 不同难度下动态增加 AI 数量与速度
- AI 卡住后的自动恢复
- AI 可拾取并自动使用道具

### 道具系统

- 赛道道具盒生成
- 玩家与 AI 可拾取道具
- 随机获得以下两种道具：
  - 加速道具：短时间额外提速
  - 冲击波：减速附近对手
- HUD 显示当前持有道具

### 幽灵车系统

- 记录玩家本局比赛轨迹
- 下一局生成半透明幽灵车
- 幽灵车不参与碰撞
- 用独立颜色区分正常赛车与幽灵车

### 环境系统

- 开始界面可选择环境：
  - 白天
  - 黄昏
  - 夜晚
- 使用光照、雾效、地面与赛道颜色变化实现不同氛围

### 成绩与存档

- 使用 `localStorage` 保存本地成绩
- 保存内容包括：
  - 最快单圈
  - 最佳总时间
- 结算界面显示历史最佳成绩
- 刷新纪录时显示“新纪录”提示

### 调试面板

- 内置开发调试面板
- 可实时调节：
  - `maxSpeed`
  - `acceleration`
  - `grip`
  - `driftFactor`
- 修改后立即作用于玩家车辆

## 操作方式

- `W` / `方向上`：加速
- `S` / `方向下`：刹车 / 倒车
- `A` / `方向左`：左转
- `D` / `方向右`：右转
- `Space`：漂移 / 手刹
- `F`：使用当前道具
- `C`：切换镜头
- `R`：手动复位

## 难度说明

开始界面可选择三档难度：

- `休闲`：AI 数量较少，整体节奏更平稳
- `进阶`：默认难度，竞争感更明显
- `极限`：AI 更多、更快，对路线和漂移要求更高

## 项目结构

```text
.
├─ index.html
├─ styles.css
├─ game.js
└─ vendor/
   └─ three.module.js
```

### 文件说明

- `index.html`
  - 页面结构
  - 开始界面
  - HUD
  - 结算界面
  - 调试面板

- `styles.css`
  - 页面布局
  - HUD 与弹层样式
  - 难度选择与环境选择样式
  - 调试面板样式

- `game.js`
  - 场景渲染
  - 赛道生成
  - 玩家与 AI 车辆逻辑
  - 比赛状态管理
  - 道具、幽灵车、环境、存档、调试系统

- `vendor/three.module.js`
  - 本地化的 Three.js 模块文件
  - 便于纯静态部署，避免依赖外部 CDN

## 主要系统模块

`game.js` 内部按职责组织，重点模块如下：

- `itemSystem`
  - 管理道具盒生成、刷新、拾取与使用

- `ghostSystem`
  - 记录玩家轨迹并在下一局进行幽灵车回放

- `weatherSystem`
  - 管理白天、黄昏、夜晚三种环境切换

- `saveSystem`
  - 读写本地成绩记录

- `debugPanel`
  - 绑定调试面板滑杆并实时更新车辆参数

## 本地运行

这是一个纯静态项目，不需要安装依赖，也不需要构建。

由于项目使用了 ES Module，直接双击 `index.html` 在部分浏览器环境下可能会遇到模块加载限制。推荐通过任意静态服务器运行，或者直接部署到 Cloudflare Pages。

如果你使用的是支持静态预览的开发工具，也可以直接把仓库根目录作为站点根目录打开。

## Cloudflare Pages 部署

项目已经适配静态部署，可直接连接仓库进行发布。

推荐配置：

- Framework preset：`None`
- Build command：留空
- Build output directory：仓库根目录

部署后默认入口文件为 `index.html`。

## 设计要点

### 为什么使用纯静态结构

- 便于快速预览和分享
- 无需 Node 构建流程
- 非常适合部署到静态托管平台

### 为什么将 Three.js 本地化

- 避免外部 CDN 不稳定导致加载失败
- 确保资源路径完全可控
- 便于 Cloudflare Pages 等平台直接部署

### 为什么采用圆角矩形赛道

- 更稳定，便于 AI 路径跟随
- 容易生成闭环中心线
- 更适合做进度投影、检查点和出界判定
